Sorting Algorithms: Bubble Sort, Quicksort, and Merge Sort

Sorting is the ordering of a list of objects. There are many types of internal sorting algorithms - the type of sorting which is done if the number of objects to be sorted is small enough to fit into the main memory (as opposed to external sorting algorithms). I will be going over 3 of the most common types: bubble sort, quicksort, and merge sort.

Bubble Sort

The bubble sort algorithm works by comparing each item against the item next to it, and swapping them if required. So, the largest element "bubbles up" to the top of the array, hence the name!

(Image courtesy of Wikimedia Commons)

So, broken down then:

  1. Compare the first two items in the list.
  2. If the first item is greater, swap the items.
  3. Now compare the second and third items in the same manner.
  4. Continue until you get to the end of the list.
  5. Repeat process n times, n being the length of your list.

Quicksort

The quicksort algorithm employs a divide and conquer algorithm. It first divides a large array into two smaller sub-arrays, consisting of the low elements and the high elements. Quicksort can then recursively sort the sub-arrays.

(Image courtesy of Wikimedia Commons)

The steps to a quicksort algorithm:

  1. Choose any element of the array to be the pivot.
  2. Divide all other elements (aside from the pivot) into two partitions:
    • All elements less than the pivot go into the first partition
    • All elements greater than the pivot go into the second partition
  3. Use recursion to sort both partitions
  4. Join the first sorted partition, the pivot, and the second partition.

Merge Sort

Merge sort is another comparison based sorting algorithm that also uses a divide and conquer strategy to sort a list. You divide the list into the smallest unit (1 element), then compare each element with the adjacent list to sort and merge the two adjacent lists. Then, all the elements are sorted and merged.

(Image courtesy of Wikimedia Commons)

Merge sort steps:

  1. Divide the unsorted list into n sublists, each containing 1 element (a list of 1 element is considered sorted).
  2. Repeatedly merge these sublists to produce new sorted sublists until there is only 1 sublist remaining. This will be sorted list!
